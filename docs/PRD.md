# Quant System - 需求文档 (PRD)

## 1. 项目概述

加密货币量化交易系统，支持分钟级策略执行，Web 可视化监控。

## 2. 目标用户
- 个人投资者（仅开发者本人使用）
- 风险承受能力：中等
- 技术水平：可配置参数，无需写代码

## 3. 功能需求

### 3.1 核心功能

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 数据收集 | 每分钟获取 OKX 行情数据 | P0 |
| 策略执行 | 多策略并行，自动根据市场状态切换 | P0 |
| 自动交易 | 满足条件自动下单，无需人工确认 | P0 |
| 风险控制 | 仓位限制、日亏损熔断、止损止盈 | P0 |
| Web 监控 | Dashboard、持仓、交易记录可视化 | P0 |
| 策略配置 | Web 页面开关策略、调整参数 | P1 |

### 3.2 策略体系

**内置策略（4种）:**
1. **TrendFollowing** - 趋势跟随（MA多头排列 + RSI 40-70）
2. **MeanReversion** - 均值回归（RSI<50 + BB下轨）
3. **Breakout** - 突破策略（突破上轨 + MACD金叉）
4. **OversoldBounce** - 超卖反弹（RSI<30）

**策略切换逻辑:**
- 根据市场状态（20日涨幅 + BB带宽）自动选择
- 强劲上涨 → TrendFollowing
- 区间震荡 → MeanReversion
- 低波动 → 观望

### 3.3 风控规则

| 规则 | 参数 |
|------|------|
| 单次最大仓位 | 70% |
| 单笔风险 | ≤15% |
| 日亏损熔断 | -6 USDT |
| 最小盈亏比 | 0.8:1 |
| 交易冷却 | 15分钟 |

### 3.4 Web 页面模块

| 页面 | 内容 |
|------|------|
| Dashboard | 总资产、今日盈亏、收益率曲线、持仓分布 |
| 持仓 | 当前持仓列表、开仓价、现价、盈亏、止损位 |
| 交易记录 | 历史交易、胜率统计、盈亏比 |
| 策略监控 | 当前策略状态、信号触发记录、策略开关 |
| 日志 | 系统日志、错误记录 |

## 4. 非功能需求

### 4.1 性能
- 数据收集：每分钟执行一次
- 策略计算：单次扫描 < 5秒
- 页面加载：首屏 < 3秒

### 4.2 可用性
- 7×24 运行（本地电脑，关机则停）
- 无需登录（单机使用）

### 4.3 安全
- API Key 环境变量存储
- 数据库本地存储
- 无外部网络暴露（仅本地访问）

## 5. 技术约束

| 项目 | 选择 |
|------|------|
| 后端 | Python 3.11 + FastAPI |
| 前端 | React + TailwindCSS |
| 数据库 | SQLite（起步）→ PostgreSQL（后期）|
| 交易所 | OKX |
| 部署 | 本地 Mac mini |

## 6. 版本规划

### MVP (v0.1.0)
- 数据收集 + 数据库存储
- 策略引擎（4种策略）
- 自动交易执行
- Web Dashboard + 持仓 + 交易记录

### v0.2.0
- 策略参数 Web 配置
- 策略绩效统计
- 风控告警

### v0.3.0
- 回测系统
- 多币种扩展

## 7. 现有系统兼容

- 现有交易机器人继续运行
- 新系统使用独立资金账户（建议）
- 后期验证稳定后再迁移

## 8. 风险声明

- 量化交易存在亏损风险
- 系统按配置自动执行，无法人工干预每笔交易
- 网络故障、API 异常可能导致交易失败
